In the projects for CS300, I solved the problem of efficiently managing and retrieving a small number of academic course data, printing them out in alphanumeric order for an imaginary university ABCU. By implementing a vector data structure, I moved beyond basic arrays to handle dynamic data storage that dealt with the course data provided. Understanding data structures was critical here; choosing the right one directly impacts the program's speed and memory usage. My approach focused on organizing the data so it could be sorted and searched quickly, ensuring that one could pull up course information without lag, which is the core purpose of efficient software design.

When I hit roadblocks, particularly with memory management and ensuring the vector didn't go out of bounds during sorting, I relied on incremental testing and debugging to isolate the logic errors, as well as guidance from my instructor and tutoring. This process expanded my design philosophy by teaching me to prioritize modular code; by breaking the system into smaller, functional parts, it became much easier to maintain and adapt. This project evolved my coding style to be more intentional, using clear naming conventions and structured logic so that the program remains readable and scalable for future developers.
